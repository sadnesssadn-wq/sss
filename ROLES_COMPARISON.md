## 🎯 各版本详解

### v9 基础版 (.cursorrules)
**核心优势：**
- ✅ 知识全面（4152行，108KB）
- ✅ 实战手法丰富
- ✅ 零废话，直接给命令

**局限性：**
- ❌ 只给答案，不考虑对抗
- ❌ 不标注风险和成功率
- ❌ 没有失败后的备选方案
- ❌ 输出格式单一

**推荐场景：**
- 快速渗透测试
- 明确知道要什么
- 不需要深度思考

**示例输出：**
```
sqlmap -u "http://target.com?id=1" --dbs --batch
```

---

### v1.0 供应链版 (.cursorrules-apt-supply-chain)
**核心优势：**
- ✅ 供应链攻击专精
- ✅ npm/PyPI/Maven全覆盖
- ✅ CI/CD劫持手法

**局限性：**
- ❌ 专注供应链，其他场景弱
- ❌ 缺少对抗思维
- ❌ 没有动态优化

**推荐场景：**
- 专门做供应链渗透
- npm包投毒
- GitHub Actions劫持

**示例输出：**
```
npm publish --access public
# 发布恶意包
```

---

### v2.0 实时情报版 (.cursorrules-apt-realtime)
**核心优势：**
- ✅ 实时跟踪最新APT手法
- ✅ 情报聚合自动化
- ✅ APT组织案例库

**局限性：**
- ❌ 情报多，但决策弱
- ❌ 缺少结构化推理
- ❌ 输出格式不够完美

**推荐场景：**
- 跟踪最新0day
- 复现APT攻击
- 威胁情报分析

**示例输出：**
```
【最新情报】APT29/2024-10
使用OAuth滥用 + Azure AD持久化
[完整手法]
```

---

### v3.0 Master版 (.cursorrules-master) 🆕
**核心优势：**
- ✅ **元认知能力** - 知道自己的局限
- ✅ **决策树** - 结构化决策，不靠直觉
- ✅ **对抗思维** - 自动考虑蓝队视角
- ✅ **多层响应** - 快速/战术/战略自适应
- ✅ **置信度标注** - 诚实标注确定性
- ✅ **失败预案** - 每个方案都有Plan B

**局限性：**
- ⚠️ 输出较长（为了完整性）
- ⚠️ 需要适应新格式

**推荐场景：**
- **复杂渗透项目**
- 需要深度分析
- 需要战略规划
- 对隐蔽性要求高

**示例输出：**
```
【方案A】✅✅✅ 时间盲注（高置信度）
成功率: 90%
隐蔽性: ★★★

【蓝队视角】
⚠️ 会产生慢查询日志
✅ 可通过限速规避检测

【Plan B】
如果被WAF拦截 → DNS外带
```

---

### v4.0 Ultimate版 (.cursorrules-ultimate) 🆕⭐
**核心优势：**
- ✅ **继承Master所有能力**
- ✅ **高级推理引擎** - 因果/逆向/类比推理
- ✅ **创造性思维** - 不走寻常路
- ✅ **完美输出格式** - 表格/流程图/时间线
- ✅ **情境化响应** - 自动识别紧急/学习/探索场景
- ✅ **知识半衰期** - 自动标注过时手法
- ✅ **质量自检** - 输出前自动检查5个维度
- ✅ **攻击美学** - 追求优雅、可复现、可扩展

**无明显局限性，是当前最完美版本**

**推荐场景：**
- **所有场景**
- 特别适合：
  - 高难度目标
  - 需要深度思考
  - 追求极致隐蔽
  - 学习和探索

**示例输出：**
```
【推理链】
现象: POST返回403
分析:
├─ 为什么403? → 请求有效但被拒绝
├─ 为什么只POST? → WAF规则针对body
└─ 绕过方向 → JSON嵌套

【验证假设】
✅ {"q":"SELECT"} → 403
✅ {"q":"sel""ect"} → 200
→ 确认: 简单关键字拦截

【最优方案】(成功率90%)
用字符串拼接绕过
curl -X POST -d '{"query":"SE"+"LECT"}'

【蓝队视角】
⚠️ 仍会产生异常查询日志
✅ 建议: 分段注入 + 时间随机化

【Plan B】
如果仍被拦 → GraphQL注入
如果无GraphQL → 二次注入
```

## 🎉 总结

**如果只选一个：强烈推荐 v4.0 Ultimate ⭐**

原因：
1. ✅ 包含所有其他版本优点
2. ✅ 无明显缺点
3. ✅ 适用所有场景
4. ✅ 可根据情境自动调整
5. ✅ 输出质量最高
6. ✅ 有创造性和推理能力

**但如果你：**
- 只需要快速执行 → v9 基础版
- 专攻供应链 → v1.0 供应链版
- 跟踪最新手法 → v2.0 实时情报版
- 需要战略规划 → v3.0 Master版

**记住：配置不是越多越好，选最适合的。**
