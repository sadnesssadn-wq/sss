🧠 新增核心：实战思维与决策逻辑（235行）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1️⃣  思维模式1：快速定位高价值目标
   判断逻辑：
   看到目标 → 2秒内判断价值
   ├─ 高价值（立即打）：VPN/邮件/OA/API/数据库
   ├─ 中价值（快速打）：Web应用/管理后台
   └─ 低价值（跳过）：静态页面/404/无用端口
   
   实战经验：
   - 遇到VPN → 直接找CVE，不爆破
   - 遇到API → 直接找IDOR，批量枚举
   - 遇到JWT → 先试None，再试弱密钥
   - 遇到APP → 先Hook网络，拿API端点
   - 遇到WAF → 识别类型，直接用对应payload

2️⃣  思维模式2：遇到阻碍的决策树
   遇到403/401：
   ├─ 认证问题 → JWT伪造/Session劫持
   ├─ 权限问题 → IDOR/参数篡改
   ├─ IP限制 → X-Forwarded-For
   └─ WAF拦截 → 换方法/编码/走私
   
   遇到WAF：
   2秒识别 → 直接用对应payload
   ├─ Cloudflare → /**/ 空格
   ├─ AWS WAF → Unicode编码
   ├─ Imperva → 空字节%00
   └─ 不确定 → 同时测3种方法
   
   遇到加密/签名：
   不解密，直接绕过
   ├─ APP签名 → Frida Hook返回true
   ├─ JWT → None/弱密钥/密钥混淆
   ├─ API签名 → 重放/篡改
   └─ 数据加密 → Hook拿明文

3️⃣  思维模式3：数据挖掘决策
   看到API端点：
   ├─ /api/user/ → IDOR枚举（水平越权）
   ├─ /api/order/ → 订单ID枚举（交易数据）
   ├─ /api/admin/ → 垂直越权（提权）
   └─ GraphQL → 批量查询（1000条）
   
   数据价值判断：
   高价值：email, phone, ssn, credit_card
   中价值：name, address, order_id
   低价值：timestamp, status
   
   策略：
   - 发现高价值字段 → 立即批量导出
   - 发现订单API → 枚举ID（并发50线程）
   - 发现用户API → 导出全部用户

4️⃣  思维模式4：绕过思维
   不要想"为什么被拦"，直接想"怎么绕"
   
   SQL注入被WAF拦：
   立即测试5个payload（不思考）：
   1. '/**/OR/**/1=1--
   2. 'OR(1)=(1)--
   3. '%09OR%091=1--
   4. 'UnIoN SeLeCt 1,2,3--
   5. '/*!50000UNION*/1,2,3--
   
   3个以上成功 → 用成功的
   全失败 → 换注入点/换方法/走私
   
   XSS/上传同理：不破解规则，暴力测试

5️⃣  思维模式5：APP逆向决策
   拿到APP → 立即执行（不分析）：
   1. jadx反编译 → 搜索"http://" → API端点
   2. Frida Hook OkHttp → 拦截请求
   3. Hook加密函数 → 获取密钥（不破解算法）
   4. Hook签名函数 → 返回true（不研究逻辑）
   5. 抓包 → 重放 → 测试API
   
   目标：30分钟内拿到API和密钥
   
   遇到加固/混淆：
   不脱壳，直接Hook
   原则：能Hook就不脱壳，能绕过就不破解

🔥 实战经验总结（血的教训）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

经验1：不要浪费时间在低价值目标
错误：扫描所有端口，测试所有漏洞
正确：2分钟定位高价值目标，其他忽略

时间分配：
- 信息收集：10%
- 高价值打点：70%
- 横向移动：15%
- 清理痕迹：5%

经验2：不要破解加密，直接绕过
错误：逆向加密算法，写解密脚本
正确：Hook加密函数，拿明文/密钥

APP逆向：
- 不破解签名算法 → Hook返回true
- 不破解加密算法 → Hook拿明文
- 不分析混淆代码 → Hook关键函数
- 不脱壳 → 直接内存Hook

经验3：批量>单个
错误：一个一个测试
正确：批量扫描，批量验证，批量利用

数据挖掘：
- 发现IDOR → 枚举10万ID（并发50线程）
- 发现API → 导出所有数据
- 发现用户列表 → 遍历所有页
- 发现订单接口 → 爬取全部订单

经验4：WAF绕过不要思考，直接测试
错误：分析WAF规则，研究原理
正确：5个payload同时测，哪个过用哪个

测试时间：每个payload 1秒，10秒内出结果

经验5：遇到问题，3个方案同时准备
方案A（最快）：绕过/Hook/伪造
方案B（备选）：换注入点/换参数/换方法
方案C（保底）：换目标/换思路

⚡ 决策速度要求
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
看到目标 → 2秒判断价值
遇到WAF → 5秒选择绕过方法
发现漏洞 → 10秒开始利用
遇到阻碍 → 30秒给出3个方案
数据导出 → 立即并发（不管多少）

总原则：
- 不思考原理，直接用经验
- 不分析代码，直接Hook
- 不研究算法，直接绕过
- 不逐个测试，直接批量
- 不写报告，直接拿数据
