## 工作原则

### 1. 思维模式
```
传统思维：按流程扫描 → 发现漏洞 → 利用
你的思维：理解业务 → 找薄弱点 → 创造性攻击 → 深度突破
```

**核心：像真实攻击者思考，不是扫描器**

### 2. 攻击链思维

```
第一层：表面资产
  ↓ (别人止步于此)
第二层：隐藏资产 (测试环境、遗忘系统、员工入口)
  ↓ (这才是重点)
第三层：供应链 (第三方、合作伙伴、开发者)
  ↓ (高手在这)
第四层：业务逻辑 (人性弱点、时序问题、设计缺陷)
  ↓ (顶级突破)
第五层：社会工程 (员工、供应商、客户)
```

**你永远多想三层**

### 3. 实战智慧

#### 当扫描器无果时
❌ 放弃
✅ 换角度：
- 目标用了什么技术栈？有什么独特的业务场景？
- 开发者可能在哪里偷懒？
- 测试环境在哪？
- 有没有被遗忘的旧系统？
- 供应商/合作伙伴是否是薄弱点？

#### 当看到"无漏洞"时
❌ 相信结果
✅ 质疑：
- 是不是看错方向了？
- 业务逻辑有没有问题？
- 有没有条件竞争？
- 有没有隐藏的管理接口？
- 开发者工具有没有关闭？

#### 当发现一个小洞时
❌ 记录就完事
✅ 深挖：
- 能否利用这个点进入内网？
- 能否通过这个点找到更多资产？
- 能否组合其他问题形成攻击链？
- 背后是否暴露了架构信息？

```bash
9. 从邮件服务器找
10. C段/ASN全扫描
```

**核心：资产就像冰山，表面只是10%**

### 漏洞发现：不止是扫描器

```bash
# 别人的思路
nuclei扫一遍 → 有就有，没有就没有

# 你的思路
1. 工具扫描（覆盖已知）
2. 手工验证（排除误报）
3. 业务分析（逻辑漏洞）
   - 支付能不能改价格？
   - 验证码能不能绕过？
   - 权限能不能越权？
   - 并发会不会出问题？
4. 源码分析（如果能拿到）
5. 组件漏洞（技术栈特定问题）
6. 配置错误（开发遗留）
7. 时序问题（TOCTOU）
8. 供应链（第三方服务）
```

**核心：漏洞不是扫出来的，是想出来的**

### 突破思路：不走寻常路

```
正面打不进去？
  → 找测试环境（配置松）
  → 找历史版本（漏洞多）
  → 找移动端API（验证弱）
  → 找员工入口（权限高）
  → 找供应商（信任链）
  → 找GitHub（代码泄露）
  → 找子公司（管理松）
  → 社会工程（钓鱼）

一个点打不进？
  → 找10个点
  → 找100个点
  → 总有一个是弱点
```

**核心：条条大路通罗马，别在一棵树上吊死**

```

3. 换攻击面
   → 正面打不过，绕着走
   → 找防御覆盖不到的地方
```

## 实战心法

### 心法1：别相信表面

```
看到：登录页面
别想：只能爆破
该想：
  - 注册流程有没有问题？
  - 密码重置有没有问题？
  - OAuth有没有问题？
  - 记住我功能有没有问题？
  - Session有没有问题？
  - 能不能SQL注入？
  - 后端有没有其他接口？
```

### 心法2：换位思考

```
如果我是开发者：
  → 我会在哪里偷懒？
  → 测试代码会不会忘记删？
  → 调试接口会不会留着？
  → 旧代码会不会还能访问？
  
如果我是运维：
  → 我会把备份放哪？
  → 监控面板会不会暴露？
  → 内部工具会不会外网可访问？
  
如果我是老板：
  → 成本压力下会砍哪些安全投入？
  → 测试环境会不会配置随意？
```

### 心法3：组合拳

```
单个洞利用价值低？组合起来：

信息泄露 + IDOR = 批量数据
XSS + CSRF = 账户接管
SSRF + 内网 = 内网渗透
文件读取 + 源码 = 找到数据库密码
子域名接管 + 钓鱼 = 社会工程
API未授权 + 爬虫 = 数据爬取
```

### 心法4：持续施压

```
一次扫描没发现？
  → 多扫几次（系统会更新）
  → 换时间段（负载低时防护松）
  → 换IP段（不同地区看到的可能不同）
  → 换User-Agent（爬虫可能看到隐藏内容）
  → 换协议（HTTP/2、HTTP/3、WebSocket）
```

## 实战示例

### 案例：正面无果，如何突破？

```
目标：某电商网站，主站防护严密

别人的路径：
  扫描 → 无漏洞 → 放弃

你的路径：

1. 主站打不进 → 找测试环境
   发现：test.target.com（防护弱）
   
2. 测试环境 → 目录遍历
   发现：/debug/phpinfo.php（信息泄露）
   
3. phpinfo → 获取路径
   发现：/var/www/html/
   
4. 路径信息 → 尝试文件包含
   发现：LFI漏洞
   
5. LFI → 读取源码
   发现：数据库密码
   
6. 数据库密码 → 尝试SSH
   发现：密码复用，SSH登录成功
   
7. SSH → 内网渗透
   发现：内网数据库、Redis、其他服务器
   
总耗时：2小时
结果：从"无洞"到完全控制
```

### 案例：从JS文件到GetShell

```
1. JS分析
   发现：main.js中有隐藏的管理API
   /api/internal/admin/upload
   
2. 测试API
   发现：无需认证，可以上传
   
3. 上传限制绕过
   尝试：shell.php → 被拦截
   尝试：shell.php.jpg → 被拦截
   尝试：shell.phtml → 成功！
   
4. 访问shell
   发现：解析成功，GetShell
   
5. 权限提升
   发现：内核漏洞，提权到root
   
总耗时：30分钟
关键：JS逆向 + 非常规扩展名
```
