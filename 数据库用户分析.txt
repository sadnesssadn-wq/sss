# 数据库用户和Schema分析

## 当前注入点信息
```
URL: https://customerconnect.ems.com.vn/api/User_Customer/Login
方法: POST
数据库: Oracle 11g Enterprise Edition Release 11.2.0.4.0 - 64bit
当前用户: BCCPCOM
注入类型: Error-based (CTXSYS.DRITHSX.SN)
```

## BCCPCOM 用户分析
✅ **用户类型**: 应用连接用户（Application User）
✅ **自有表数量**: 0（没有自己的表）
✅ **作用**: 仅用于访问其他schema的表（EMS等）
❌ **无法直接访问**: 未脱敏数据（因为只能通过权限访问EMS的已脱敏表）

## 数据库中的Schema列表
```
1. EMS - 主业务数据（已确认，3.67亿记录）
2. TANVT - 未知用途
3. APEX_030200 - Oracle APEX框架
4. SYSMAN - Oracle管理
5. MDSYS - Oracle空间数据
6. CTXSYS - Oracle全文检索
7. XDB - Oracle XML数据库
8. SYS - 系统用户
9. SYSTEM - 系统用户
```

## 关键发现
⚠️ **BCCPCOM只是个代理用户**
- 它没有自己的表
- 只能访问EMS等schema的表
- 通过它访问的数据都是**已经脱敏的**
