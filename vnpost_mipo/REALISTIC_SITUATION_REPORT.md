## 问题分析

### Hydra检测逻辑的局限性

```bash
# 当前使用
F=login  # 如果响应不包含"login"就算成功
F=login_box  # 如果响应不包含"login_box"就算成功

# 问题
ASP.NET的响应可能:
1. 登录失败也不包含这些字符串
2. 重定向导致Hydra误判
3. Session管理问题
```

### 为什么会误报

VNPost的系统特点：
- 复杂的ASP.NET ViewState机制
- 可能有二次验证
- 响应不一致
