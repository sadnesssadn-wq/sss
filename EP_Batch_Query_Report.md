## 🎯 API使用统计

### 使用的API
1. **DeliveryLadingJourney** - 第一优先级
   - 用于查询已配送信息
   - 返回最详细的配送数据
   
2. **Inquiry** - 第二优先级（作为备用）
   - 用于查询未配送包裹
   - 返回基本信息和联系方式

### 查询策略
```
1. 先调用 DeliveryLadingJourney（不需要签名，最快）
   └─ 如果返回数据 → 包裹已配送，获取完整信息
   └─ 如果返回"未找到" → 继续下一步

2. 调用 Inquiry（需要签名）
   └─ 如果返回数据 → 包裹未配送，获取基本信息
   └─ 如果返回"未找到" → 运单号不存在
```

这个策略**效率最高，信息最全**！
