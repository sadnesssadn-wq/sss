## 📋 社工话术（3个场景）

### 场景1: 新员工身份 ⭐⭐⭐⭐⭐

**开场白：**
```
你好！我是贵公司新入职的后端开发。
今天第一天上班，HR让我先熟悉一下公司的系统。

我需要访问API文档，请问Apifox的链接是什么？
我的mentor今天请假了，找不到人问。
```

**预期回复1：** "请联系你的mentor或HR"
**应对话术：**
```
我已经发邮件给mentor了，但他说今天有急事。
HR说技术问题问技术支持。
我只是需要API文档的链接，应该是公开的吧？
或者你能给我技术负责人的联系方式吗？
```

**预期回复2：** "我们没有Apifox"
**应对话术：**
```
哦对不起，可能我记错了。
那我们公司用什么管理API文档？Swagger？还是自己搭的？
访问地址是什么？
```

**预期回复3：** "我不清楚，建议发邮件给xxx@lightnode.com"
**成功！** 获得了内部邮箱格式和可能的员工姓名

---

### 场景2: 技术支持身份 ⭐⭐⭐⭐

**开场白：**
```
你好，我是客户XXX公司的技术对接人。

我们在集成贵司API时遇到问题。
你们的技术文档（Apifox）上的某个接口调用总是返回401。

能否帮我确认一下：
1. 我访问的文档链接是正确的吗？（https://s.apifox.cn/project-XXXXX）
2. 还是有新的文档地址？
```

**预期回复1：** "这个链接不对"
**应对话术：**
```
那正确的链接是什么？
或者你能帮我转接给负责API对接的技术同事吗？
他的邮箱是多少？
```

**预期回复2：** "请发邮件给support@lightnode.com"
**应对话术：**
```
我已经发了，但24小时没回复。
这个问题很紧急，今天必须解决。
能给我负责这块的技术负责人的直接联系方式吗？
比如他的工作微信或者直线电话？
```

---

### 场景3: 合作伙伴身份 ⭐⭐⭐⭐

**开场白：**
```
你好，我是XX云服务商的商务。

我们和贵司技术团队有个合作项目在进行中。
技术负责人张工（或李工、王工）之前给我们发过API文档。

但我这边电脑重装了，Apifox链接找不到了。
能麻烦你帮我问一下技术部门，把链接再发我一次吗？

或者直接给我张工的邮箱，我自己联系他。
```

**关键点：**
- 故意说错名字（张工、李工），让客服纠正
- 如果客服说"我们没有张工"，立即道歉说"不好意思，可能是王工"
- 目标是让客服说出真实的技术负责人姓名

## 🎭 高级技巧

### 技巧1: 利用时间压力
```
"这个问题很紧急，老板在催。"
"客户那边在等，如果今天解决不了会影响合作。"
"我现在就在会议室，领导在旁边等我。"
```

### 技巧2: 表现专业性
```
不要问"你们用什么技术"
而是说"你们用的是Apifox还是Swagger？"

不要问"API文档在哪"
而是说"我访问 https://s.apifox.cn/project-12345 返回404，是不是项目ID变了？"
```

### 技巧3: 建立信任
```
"我之前和你们的XX（某个员工）对接过。"
"我在你们官网看到了XX技术栈，很不错。"
"我们公司也在用LightNode的VPS，体验很好。"
```

### 技巧4: 逐步提问
```
不要一次问太多问题！
第一轮：只问一个简单问题
第二轮：根据回复继续深入
第三轮：请求联系方式

每个问题之间要有礼貌的感谢和寒暄。
```

## ⚠️ 注意事项

### 不要做的事
```
❌ 不要表现得太着急（会引起怀疑）
❌ 不要一次问太多问题（会被拒绝）
❌ 不要说自己是"安全测试"（立即被挂断）
❌ 不要威胁或恐吓（触犯法律）
❌ 不要录音或录屏（隐私问题）
```

### 要做的事
```
✅ 表现得自然、礼貌、专业
✅ 使用他们的术语（VPS、CDN、API）
✅ 适时表达感谢
✅ 如果被拒绝，礼貌结束对话
✅ 详细记录所有信息
```
