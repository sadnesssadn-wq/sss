# 🎯 真正有效的APT钓鱼话术

## 核心原则

```
❌ 假的话术特征：
- "系统升级"
- "安全验证"
- "紧急通知"
- 正式口吻
- 过度紧急

✅ 真实话术特征：
- 具体业务问题
- 日常工作场景
- 轻松自然语气
- 有上下文
- 可验证的细节
```

---

## 🔥 话术1：生产事故（VPN/内网）

### 目标：运维/技术负责人

**主题：** 香港节点客户VPS大面积无法SSH连接

**正文：**
```
@运维组

香港HK1机房从今天下午2点开始，有客户反馈VPS无法SSH连接。

已确认问题：
• 受影响VPS：约200+台
• 报障客户：15+
• 节点：hk1-compute-03.lightnode.com
• 症状：SSH超时，VNC黑屏

客户工单已经炸了，我在外地出差，电脑没带。

能帮忙登VPN进去看下吗？
排查下是不是母机网络出问题了。

VPN临时账号我发工单申请了，但IT那边还没批。
你们用自己的VPN先上去看看？

母机IP：10.20.3.47

麻烦了🙏

---
李明 | 运维
发自手机
```

**为什么有效：**
- ✅ 真实业务场景（VPS故障确实常见）
- ✅ 具体数字（200台、15个客户）
- ✅ 内网IP（10.20.3.47看起来真实）
- ✅ "我在外地"→合理化自己不能处理
- ✅ 非正式语气（"工单炸了"、"麻烦了🙏"）
- ✅ 不要求对方点链接，只是"求助"

**钓鱼方式：**
- 不发钓鱼链接
- 等对方回复"我没VPN"或"VPN登不上"
- 再回复："那你试试这个临时通道：https://vpn-temp.lightnode.com"

## 🔥 话术2：客户紧急需求（诱导登录内部系统）

### 目标：技术支持/开发

**主题：** Re: 企业客户API限流问题

**正文：**
```
Hi团队，

客户（北京某游戏公司）刚打电话过来，说他们API调用一直被限流。

客户信息：
账号ID：LN-ENT-8472
套餐：企业版（月付$2000+）
问题：API限流（实际QPS只有10，但报错429）

说如果今晚解决不了就退款+投诉。

我查了后台，看不到详细限流日志。
你们谁有API网关的权限？帮忙查下？

如果要我登录查，发我个临时账号？
或者把限流规则截图给我也行。

客户电话：138-xxxx-xxxx（一直在催）

急！

---
王经理 | 商务支持
```

**为什么有效：**
- ✅ "Re:"开头（像是邮件回复，有上下文）
- ✅ 具体客户（游戏公司）+金额（$2000+）
- ✅ 紧急但合理（上新游戏确实紧急）
- ✅ "我查不到"→诱导对方登录
- ✅ 提供选择（"临时账号"或"截图"）

**钓鱼方式：**
- 对方可能直接给你截图（获得内网信息）
- 或者对方说"我给你开个临时账号"→你提供钓鱼登录页
- 或者对方说"你用这个账号登"→直接拿到凭证

```
猎聘网 | 高级猎头
手机：186-xxxx-xxxx

```

**为什么有效：**
- ✅ 猎头推荐很常见，HR不会怀疑
- ✅ "目前在职"→解释为什么Word格式（公司内网）
- ✅ 背景吸引人（阿里云P7、CTO）
- ✅ 薪资合理（60-80w）
- ✅ 非正式语气（"你看看合适不"）

**宏木马：**
- Word文档伪装成简历
- 宏代码隐藏在"格式设置"中

- 诱导启用宏："此简历包含动态图表，请启用编辑查看"

## 🎯 执行策略

### 阶段1：建立信任（不发钓鱼链接）

**Day 1：发"求助"邮件**
```
发话术1（生产事故）到 ops@/tech@
内容：只描述问题，不发链接
目的：看谁回复，建立对话
```

**Day 2：跟进对话**
```
对方回复后，继续聊业务问题
展现专业性（谈技术细节）
不急着钓鱼
```

**Day 3：自然引入**
```
"你能登VPN看下吗？"
对方说"可以" → 不钓了，换话题
对方说"我没VPN" → 提供"临时通道"
```

### 阶段2：投放木马（不直接说是软件）

**发话术3（猎头推荐）到 hr@**
```
附件：真实的简历Word（带宏）
简历内容专业（从LinkedIn复制）
宏隐藏在"格式设置"中
```

### 阶段3：批量覆盖（多点开花）

**同时执行：**
```
话术1 → ops@/tech@（3封）
话术3 → hr@（1封）
话术4 → support@（1封）
话术6 → business@（1封）
```

## 🔥 关键技巧

### 1. 不要一次发完
```
❌ 今天发21封邮件 → 太明显
✅ 每天2-3封，持续7天 → 自然
```

### 2. 先聊天，再钓鱼
```
❌ 第一封就发钓鱼链接 → 假
✅ 聊2-3轮，建立信任再发 → 真
```

### 3. 提供选择
```
❌ "点这个链接"
✅ "你能帮忙查吗？查不了的话我再想办法"
```

### 4. 用真实业务术语
```
❌ "数据库"
✅ "hk1-compute-03母机"
✅ "API网关限流规则"
✅ "VPS控制台"
```

### 5. 非正式语气
```
❌ "请您协助处理"
✅ "能帮个忙吗？🙏"
✅ "太感谢了😄"
✅ "客户工单炸了"
```
