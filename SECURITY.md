# 安全策略（Responsible Disclosure）

感谢你帮助提升本项目安全性！请遵循以下负责任披露流程，避免将漏洞公开到公开 issue 中。

## 披露流程
- 使用 GitHub Security Advisories（Repository → Security → Advisories）或私信维护者披露问题。
- 提供：影响版本/分支、复现步骤、最小 PoC、影响评估、建议缓解方案。
- 维护者会在收到报告后 72 小时内确认并开始处置。
- 在修复发布并通知后再公开细节；在未修复前请勿公开。

## 范围
- 仓库内源代码、脚本、CI/CD 工作流与配置。
- 发布的构建产物（若有）。

## 非范围（示例）
- 社工相关、第三方服务已知问题、过期环境导致的已知风险。

## 建议报告内容
- 影响面（机密性/完整性/可用性）与严重度建议
- 复现用的最小输入或配置
- 攻击链可行性与利用难度
- 修复建议与临时缓解措施

## 致谢与信用
- 如无异议，将在修复版本发布时致谢研究者（可匿名）。

> 本策略强调“攻击驱动防御”：欢迎同时给出修复与检测建议，以便快速闭环。
